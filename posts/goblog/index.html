If you would’ve Googled me ten years ago, you probably would’ve ended up on my personal blog. Slowly this personal blog got replaced by all sorts of (social) channels. Facebook. Tumblr. Medium. And website builders like Squarespace. I don’t mind using these channels and services, but I regret neglecting and eventually ditching my blog. I think it’s healthy to keep some sort of archive. And it’s fun right, having this place on the web that’s really yours. I appreciate initiatives like IndieWeb, a community of personal websites that own their domain, data and even use it as a primary identity online.

I love how I can still browse to Zeldman.com or Andy Budd’s blog and find this immense archive. I also think that having your own space has a lower threshold to post stuff that interests you. It’s less influenced by likes, claps or retweets. Adactio has more personality than any social media profile I can think of.

It’s weird, I miss the old web, while it’s still here. Instead of regretting choices of the past, it was time to make new choices. I don’t plan on blogging a lot, but at least what I write can be found on something I own.

Go with Hugo

With that in mind I had to decide on how to run this blog. I have a lot of experience with Wordpress and other Content Management Systems, but wanted to try something new. In the past I’ve worked with static site generators, like Middleman and Jekyll (both are based on Ruby). There is no database or server-side stuff needed when you publish. This has another advantage, you can host the website as a https://pages.github.com.

I decided to go for Hugo, which runs on Go. Hugo is well maintained and I wanted to know a little bit more about Go (although you actually don’t really need to learn Go, to use Hugo).

Just like most of the static site generators, it uses Markdown files to generate blog posts as static html. You can add a CMS with a service like Forestry. Even if Hugo isn’t the right fit, it doesn’t matter. I can just copy or export the Markdown files to an alternative. I hope this makes it more future proof than previous websites.

Now, let’s see if we can still read this blog post in 2028.
